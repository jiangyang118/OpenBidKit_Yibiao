# 易标投标工具箱用户手册与故障排查

本文用于给用户、测试人员和后续 Codex 快速确认核心功能入口、交付物和常见问题处理方式。本文只描述当前已落地能力；MinerU 精准解析真实网络回归作为可选增强仍按 `plan.md` 后续迭代。

## 1. 技术方案与已有方案扩写

- 入口：左侧主菜单 `生成技术方案`、`已有方案扩写`。
- 输入：招标文件；已有方案扩写额外导入原方案。
- 核心流程：招标文件解析 -> 招标要求分析 -> 目录生成/编辑 -> 全局事实 -> 正文生成/编辑。
- 交付：正文 Markdown 预览、图片/ Mermaid 预览、Word 导出；导出格式页可配置封面、目录页、一级章节分节、页眉、页脚、标题样式、正文样式、表格样式、图片最大宽度和文字水印。
- 注意：目录重新生成、编辑、添加或删除后，旧正文内容和正文生成缓存会被清空，避免目录与正文错位。

常见问题：

- 导入文件失败：先到 `设置 -> 文件解析` 检查本地解析 / 本地 OCR / MinerU 配置；扫描件、图片优先走本地 OCR，本地 OCR 默认使用 PaddleOCR。OFD 会先尝试本机 OFD 转 PDF 工具或 LibreOffice/WPS 转 PDF 后 OCR；如果转换工具不可用，请先手动另存为 PDF 再导入。
- Word 图片缺失：导出会给出 warnings；缺失本地图片不会阻断 docx 生成，请按 warnings 回到正文图片来源修复。
- Word Mermaid 图转换失败：导出会插入替代图并给出 warnings，请按提示检查 Mermaid 语法或网络连接后重新导出。
- 任务中断：重新进入页面后先读取 Store 和 active task；如任务已无 active task，应重新执行对应步骤。

## 2. 商务标

- 入口：`标书制作 -> 商务标`。
- 输入：可复用技术方案招标文件，也可独立导入商务标招标文件。
- 核心能力：商务响应矩阵、合同偏离、资信材料、报价附件、负责人/确认人、AI 结构化提取、独立附件管理。
- 交付：Markdown、Word `.docx`、Excel `.xlsx` 商务响应交付包。

常见问题：

- AI 结构化提取失败：系统保留已有矩阵，不会清空；可继续使用规则提取结果并人工编辑。
- 附件删除后仍显示：刷新页面读取 Store；如文件本体已被清理，重新导入附件。

## 3. AI 评标

- 入口：`标书制作 -> AI 评标`。
- 输入：评分办法/招标文件、投标文件。
- 核心能力：评分项抽取、投标文件证据匹配、多投标文件评分、批量重评、专家打分、审计意见、评标委员会会议纪要模板。
- 交付：Markdown 自评报告、Word `.docx` 正式报告、Excel `.xlsx` 多工作表报告、Word `.docx` 会议纪要模板。

常见问题：

- 投标文件证据缺失：检查导入的 Markdown 原文是否包含评分项关键词；证据字段会尽量携带章节、行号和命中关键词。
- 专家分差异常：系统会生成专家间分差和专家均分偏差审计意见，需人工复核后调整最终分。

## 4. 投标机会

- 入口：左侧主菜单 `投标机会`。
- 输入：手动录入公告、导入公告文件或输入公告 URL。
- 核心能力：规则/AI 结构化解析、机会评分、经营维度拆解、负责人、下一步动作、提醒时间、多轮跟进记录、公告/沟通附件、知识库匹配。
- 交付：Markdown 投标建议报告、`.ics` 日历提醒。

常见问题：

- URL 导入内容为空：网站可能反爬或需要登录；可保存网页为文件后导入。
- 附件导入后找不到原文件：系统会复制到当前工作区 `bid-opportunity/attachments/`，报告中展示工作区路径和原始路径；删除附件会同步清理复制文件。
- 跟进信息和报告不一致：保存跟进记录或附件备注后重新导出，投标建议报告以 Main 侧 Store 当前状态为准。
- 日历导出缺少机会：只有设置了提醒时间的机会会进入 iCalendar。

## 5. 知识库与图片知识库

- 文档知识库入口：`知识库`。
- 图片知识库入口：`资源与素材 -> 图片知识库`。
- 文档知识库能力：文档迁移、解析、分析、匹配、active task 恢复。
- 图片知识库能力：上传、分类、文件夹、标签、批量管理、正文插入、引用回溯、正文生成优先复用。

常见问题：

- 旧知识库迁移提示：只迁移旧版 `status = success` 的文档；未完成或处理中旧文档不会迁移。
- 图片无法导出到 Word：确认图片仍存在于本地工作区，正文引用应为 `yibiao-asset://image-knowledge-base/...` 或可解析的本地资源。

## 6. 标书查重

- 入口：`标书检查 -> 标书查重`。
- 输入：可选招标文件，多份投标文件。
- 核心能力：元数据、目录、正文重复句、完全重复图片、感知 hash 相似图片；相似图片覆盖压缩、缩放、旋转、右下角水印/角标提示、主体内容裁剪框和 16x16 细粒度局部截图裁剪框。
- 人工处理：确认重复、忽略、恢复待处理、批量处理、正文常用忽略规则。
- 交付：Markdown / Word / PDF 查重报告，包含批量处理建议、人工状态、重复/相似图片证据、相似图片复核视图和视觉证据元数据；PDF 会输出图形化复核卡。

常见问题：

- 招标文件引用句被误判：导入招标文件后会作为白名单过滤；也可把固定模板句加入正文忽略规则。
- 相似图片没有命中：当前检测基于感知 hash、尺寸比例和有限旋转/裁剪/水印启发式；过度重绘、主体变化太大或截图质量极差时仍需人工复核。
- 报告内容和页面不一致：确认/忽略/删除后重新导出报告，报告以 Main 侧 Store 当前状态为准。

## 7. 废标项检查

- 入口：`标书检查 -> 废标项检查`。
- 输入：招标文件、投标文件；可从技术方案读取招标文件。
- 核心能力：无效与废标项解析、废标风险检查、错别字检查、逻辑谬误检查、单投标文件重查。
- 人工处理：忽略、恢复、删除、批量处理、复制证据。
- 交付：Markdown / Word / PDF 废标项检查报告，包含证据定位索引、章节/行号/前后文提示、文本型证据截图视图、页面截图候选匹配和真实裁剪图；本地 PDF、DOCX、DOC、WPS 和 MinerU 返回图片均可生成页面截图候选。Markdown 会引用裁剪图，Word 会嵌入裁剪图，PDF 会保留裁剪图资产引用和图形化证据卡。

常见问题：

- 证据无法精确定位：报告会保留原文证据和人工检索提示；请回到对应投标文件正文搜索。
- 只想重查一份投标文件：在结果页筛选到具体投标文件后使用“重新检查当前投标文件”。

## 8. 设置、资源与开发者工具

- 设置：配置文本模型、生图模型、文件解析、主题、布局、导出格式、更新渠道。
- 资源下载：读取 Analytics 资源接口；接口失败时优先展示本地缓存，仍无缓存时展示友好空态。
- 开发者工具：Prompt 调试台、文件解析沙盘、导出预演、JSON 请求实验室。

常见问题：

- 本地模型没有 API Key：LM Studio / Ollama 等本地服务允许无 API Key；先确认 Base URL 和模型列表。
- 资源加载失败：检查网络或服务端资源接口；客户端失败不影响其他主流程。

## 10. 已知边界

- 设置页已提供项目工作区列表、新建并切换、切换确认、归档/恢复、删除、复制、导出项目包和导入项目包入口；切换或删除前会阻止后台任务运行中的操作。切换项目后当前会话会热刷新工作区服务，后续业务数据从新项目工作区读取。
- 查重和废标检查均已支持 Markdown / Word / PDF 报告；查重报告已包含复杂相似图视觉证据和 PDF 图形化复核卡，废标项检查报告已包含文本型证据截图视图、页面截图候选匹配、真实裁剪图生成，并支持本地 PDF、DOCX、DOC、WPS 与 MinerU 返回图片的页面截图候选。
- 文件解析能力已有支持矩阵、解析沙盘、本地真实样本文档回归和本地 OCR 回归；扫描 PDF / 图片可通过本地 OCR 解析并生成页面截图候选，本地 OCR 优先使用 PaddleOCR，共享运行时不可用时回退到 Tesseract。MinerU-Agent 真实网络解析回归已通过；MinerU 精准解析作为可选增强，可在具备 Token 和网络的环境中运行 `YIBIAO_RUN_MINERU_E2E=1 YIBIAO_MINERU_TOKEN=<token> npm run test:unit -- fileServiceParserRegression.test.ts` 后补跑固化 zip 图片资产和页面截图候选结果。
