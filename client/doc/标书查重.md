# 标书查重

本文记录当前 `标书查重` 模块的已落地能力和后续开发约束。早期“先 mock 前端展示”“后续接入查重任务”等说明已过期，当前页面已经接入真实 Main 侧查重任务和 SQLite Store。

## 入口与范围

- 入口：`标书检查 -> 标书查重`。
- 可选输入：1 份招标文件，用于过滤多份投标文件共同引用的招标原文。
- 必选输入：多份投标文件。
- 分析维度：元数据、目录、正文重复句、图片重复/相似。

## 当前能力

- 元数据：提取投标文件元数据并横向对比，重复或异常项在结果表中提示。
- 目录：抽取多份投标文件目录，识别完全相同或高度相似的目录结构。
- 正文：按句子归一化比对，过滤招标文件引用句，展示重复句、涉及投标文件和出现次数。
- 图片：支持精确 hash 重复图和感知 hash 相似图；相似图已覆盖压缩、缩放、旋转、右下角水印/角标提示、主体内容裁剪框和 16x16 细粒度截图局部裁剪框。
- 人工处理：正文重复句和图片组支持确认重复、忽略、恢复待处理、批量确认、批量忽略、批量删除。
- 忽略规则：正文重复句可保存为常用忽略规则，规则带分类，支持 JSON 导入/导出并可跨项目复用。
- 报告导出：支持 Markdown、Word `.docx`、PDF 报告；报告包含批量处理建议、人工处理状态、正文忽略规则、重复/相似图片证据、相似图片复核视图、视觉证据元数据和 PDF 图形化复核卡。

## 架构约束

- Renderer 只负责文件选择、启动任务、订阅事件、读取 Store 快照和展示结果。
- 查重分析在 Electron Main 后台任务中执行，入口为 `tasks.startDuplicateAnalysis()`。
- 结果状态保存在 SQLite，运行时表结构以 `client/electron/services/sqliteDatabase.cjs` 为准，目标结构同步在 `sql/workspace_schema.sql`。
- 报告导出通过 `duplicateCheck.exportReport({ format })` 进入 Main 侧，Markdown 明确使用 UTF-8 写入，Word/PDF 复用同一份报告内容。
- 新增正文规则、图片证据字段或报告字段时，需要同步 Store、IPC/preload 类型、页面展示、报告导出和测试。

## 验证建议

修改查重模块后优先运行：

```powershell
cd client
node --check electron\services\duplicateCheckService.cjs
node --check electron\services\duplicateCheckStore.cjs
npm run test:unit -- duplicateCheckStore.test.ts DuplicateCheckPage.test.tsx
npm run build
npm run test:e2e
```

若只改文档，可至少运行 `npm run test:unit -- DuplicateCheckPage.test.tsx` 和 `npm run build` 确认页面引用未回归。
