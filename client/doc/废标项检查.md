# 废标项检查

本文记录当前 `废标项检查` 模块的已落地能力和后续开发约束。早期占位页面说明已过期，当前页面已经接入真实 Main 侧解析、检查、持久化和报告导出链路。

## 当前能力

- 客户端入口：主菜单 `废标项检查`。
- 支持导入招标文件和投标文件，也可复用技术方案 Step02 已解析出的无效投标与废标项。
- 招标文件解析会明确区分“无效投标”和“废标项”，并支持用户维护自定义检查项。
- 检查任务由 Electron Main 后台执行，页面只负责启动任务、订阅状态、读取 Store 和展示结果。
- 检查结果覆盖废标风险、错别字和逻辑谬误；废标项检查默认开启，错别字和逻辑谬误可配置。
- 结果支持按投标文件筛选、单项忽略、恢复、删除，以及批量忽略、批量恢复、批量删除。
- 支持针对当前投标文件重新检查，只替换目标投标文件结果，保留其他投标文件结果和人工处理状态。
- 导入文件时会保存解析图片资产和页面截图候选；本地 PDF 可渲染页面 PNG，DOCX/DOC/WPS 会 best-effort 转 PDF 后生成页面候选；MinerU 返回的图片也会进入页面截图候选。
- 报告导出支持 Markdown、Word `.docx` 和文本型 PDF，三种格式复用同一份报告内容。

## 证据定位与报告

- 报告包含招标/投标文件范围、无效与废标项解析状态、废标风险、错别字、逻辑谬误和处理建议。
- 报告只统计未忽略结果，并记录忽略数量。
- 证据定位明细覆盖投标文件、位置线索、原文/证据、原因、建议、章节、行号、匹配文本和前后文片段。
- 每条废标风险、错别字和逻辑问题都有稳定 HTML 锚点，并在索引表中提供跳转链接。
- 文本型“证据截图视图”会标记目标行和前后文，并进入 Markdown、Word 和 PDF。
- 报告导出时可从页面截图候选源图生成真实裁剪 PNG；Markdown 引用裁剪图，Word 嵌入裁剪图，PDF 保留裁剪图资产引用并继续绘制图形化证据卡。

## 后续约束

- Renderer 不直接保存业务权威状态，不直接读写文件，不执行耗时 AI 检查。
- IPC 只做通道注册和参数转发；检查、导入、报告生成逻辑保持在 Main 侧 Store / Task / Service。
- Markdown 报告必须显式使用 UTF-8 写入；Word/PDF 继续复用同一份 Markdown 报告内容，避免多套报告口径分叉。
- 判断材料缺失时，必须优先识别章节标题、目录项、附件标题、材料清单、表格条目、页码线索和图片占位线索；不能仅因扫描件正文不可见就判定材料缺失。
- 签字、盖章等纸质文件事项不应作为电子投标文件的确定性废标判断，只能作为人工复核提示。
- 删除业务资源时，需要同步清理导入图片资产和裁剪图资产。
- 新增结果类型或报告字段时，必须同步 SQLite schema、IPC/preload/types、页面展示和报告测试。
