# 资源下载

本文记录当前 `资源下载` 模块的已落地能力和后续开发约束。早期静态样式稿说明已过期，当前客户端已经从 Analytics 资源接口读取真实资源列表。

## 当前能力

- 客户端入口：主菜单 `资源下载`。
- 数据来源：Analytics Worker 的公开资源接口。
- 页面支持搜索资源标题、简介、标签和弹窗内容摘要。
- 资源卡片展示标题、简介、标签、累计点击量和封面。
- 资源没有图片时，客户端使用稳定的彩色封面占位，不要求管理端强制上传图片。
- 点击资源后打开详情弹窗，展示 Markdown 资源说明、下载方式或外部链接。
- 成功读取资源后会把公开资源列表缓存到 Renderer `localStorage`；接口失败时优先展示离线缓存，没有缓存时展示友好空态。
- 点击资源时通过 `trackResourceClick()` 上报 `resource_click`，只上报稳定资源 key，不上传用户文件、本地路径或正文内容。

## Analytics 管理端

- Analytics Dashboard 已提供资源管理表格，展示图片、标题、标签、Markdown 内容摘要、启停状态、排序、累计点击量、编辑和删除入口。
- Worker 侧资源 API 支持管理端鉴权、新增资源、标签去重、R2 图片读取、公开列表搜索、禁用资源隐藏、删除资源时清理图片。
- 点击量口径为资源表历史累计点击量叠加 Analytics Engine 当天 `resource_click`，Cron 会把历史 AE 点击累加回资源表。
- Worker / Dashboard 部署前通过 `npm run check` 做 `node --check` 语法验证。

## 后续约束

- 资源下载页不得回退为静态假数据；接口失败只能走本地缓存或空态。
- R2 图片是可选字段；无图占位必须保持稳定。
- 新增资源字段时，需要同步 Worker API、Dashboard 表格、客户端类型和测试。
- 埋点只允许稳定低基数字段，不得上传资源下载链接之外的用户本地信息、Prompt、正文草稿或错误详情。
- 修改 Analytics 资源管理能力时，必须保留新增、编辑、删除、图片展示、Markdown 弹窗、累计点击量和公开列表搜索能力。

## 验证

- `ResourcesPage.test.tsx` 覆盖成功加载、缓存写入、接口失败缓存回退、无缓存友好空态和资源点击埋点。
- `analytics/worker` 测试覆盖资源 API 主流程、R2 图片、公开列表搜索、禁用隐藏、删除清理和累计点击量。
- `analytics/dashboard` 测试覆盖资源管理表格图片、标签、累计点击量、Markdown 摘要、启停状态和编辑/删除入口。
- Playwright E2E 覆盖主导航进入资源下载页、模拟资源展示、无图占位和 Markdown 详情弹窗。
